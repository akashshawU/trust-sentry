CP1 Intent Guard
Evaluates BEFORE the AI agent receives the request
What it intercepts

Every incoming request before the AI agent ever sees it. The agent receives nothing until CP1 approves.

Three-tier evaluation
⚡
Tier 1 — Definitive Rules (instant)
26+ injection patterns, financial crime, regulatory hard stops, child safety. Result in <5ms. Zero API calls.
🧠
Tier 2 — Groq AI (contextual)
llama-3.1-8b-instant evaluates intent, bias, compliance, and access in a single structured JSON call. Catches zero-keyword social engineering attacks.
🛡
Tier 3 — Enhanced Rules (fallback)
If AI is unavailable, extended rule library covers 200+ patterns across all pillars. Always on.
What it checks
• Caller identity and role
• Bias in task instructions
• Agent type and permissions
• Compliance framework violations
• 13 proxy guardrail rules
• Cross-domain access attempts
• Prompt injection (OWASP LLM01-10)
• Bulk data export patterns
• Social engineering patterns
• After-hours autonomous access
• Authority impersonation
• Unknown agent detection
Outcomes
🔴 BLOCK — Agent never receives the request. Zero data accessed. Full incident logged.
🟠 ESCALATE — Request held for senior approval. Nothing executes until human decides.
🟢 ALLOW — Request forwarded with any applicable restrictions noted.

When CP1 fires
Caller is authorised, agent type appropriate, task description clean, all 13 guardrail rules pass, no bias or compliance violations detected.
When CP2 fires
Output contains no PII outside permitted scope, no compliance violations in content, agent stayed within CP1-approved boundaries.
